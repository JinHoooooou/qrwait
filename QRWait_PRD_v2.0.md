## 1. 제품 개요

QR 웨이팅 서비스는 식당 방문 고객이 **별도 앱 설치 없이** QR 코드 스캔만으로 웨이팅을 등록하고, 점주는 **별도 기기 없이 브라우저**에서 실시간으로 대기 현황을 관리할 수 있는 웹 기반 서비스입니다.

Phase 1에서 손님(B2C) 경험을 완성했고, Phase 2에서는 **점주(B2B) 운영 경험**을 구축합니다.

| 항목            | 내용                               |
|---------------|----------------------------------|
| 서비스명          | QR Wait (가칭)                     |
| 서비스 유형        | 웹 애플리케이션                         |
| Phase 1 타겟    | 식당 방문 손님 (B2C) ✅ 완료              |
| Phase 2 타겟    | 단일 매장 직접 운영 점주 (B2B)             |
| 핵심 가치         | 점주: 기기 구매 없이 회원가입 후 즉시 웨이팅 운영 가능 |
| Phase 2 제외 범위 | 다중 매장 관리, SMS/카카오 알림, 결제, 직원 계정  |

---

## 2. 문제 정의 (Problem Statement)

### Phase 1에서 해결한 문제

- ✅ **손님:** 앱 설치 없이 QR 스캔 한 번으로 웨이팅 등록
- ✅ **손님:** 자유롭게 이동하면서 실시간으로 대기 순서 확인
- ✅ **점주:** 별도 기기 없이 QR 출력만으로 서비스 도입 가능

### Phase 2에서 해결할 문제

Phase 1 이후 서비스가 실제로 운영되려면 점주가 웨이팅을 직접 관리할 수 있어야 합니다. 현재는 다음과 같은 치명적 공백이 있습니다.

- 입장 처리 UI가 없어 **대기번호가 영원히 줄어들지 않음**
- 노쇼 손님을 처리할 방법이 없어 **대기열이 막힘**
- 영업 종료 시 신규 웨이팅 등록을 막을 수 없음
- 누구나 아무 매장이나 접근 가능한 **보안 공백** 존재
- 예상 대기시간이 `팀수 × 5분`으로 **하드코딩**되어 매장마다 부정확

## 3. 목표 및 성공 지표

### Phase 2 제품 목표

| 목표          | 세부 내용                   | 우선순위 |
|-------------|-------------------------|------|
| 점주 인증       | 회원가입/로그인으로 본인 매장만 관리 가능 | P0   |
| 대기 목록 관리    | 실시간 대기 현황 조회 및 입장·노쇼 처리 | P0   |
| 웨이팅 상태 제어   | 브레이크타임·영업종료·만석 상태 전환    | P0   |
| 예상 대기시간 정확도 | 점주가 직접 설정한 회전 시간 기반 계산  | P1   |
| 점주 알림       | 대기자 급증 시 점주에게 알림 발송     | P1   |

### Phase 2 성공 지표 (KPI)

- 점주 회원가입 후 첫 QR 발급까지 소요시간 **5분 이내**
- 점주의 입장 처리 소요시간 **3초 이내** (버튼 탭 → 처리 완료)
- 웨이팅 마감 설정 후 손님 화면 반영까지 **1초 이내**

## 5. 기능 요구사항 (Functional Requirements)

### FR-01. QR 스캔 후 웨이팅 등록 (Phase 1, 기존 유지)

- QR 코드에는 매장 고유 storeId가 인코딩되어 있음
- 스캔 시 별도 앱 없이 브라우저에서 웨이팅 등록 페이지로 이동
- 입력 항목: 이름(닉네임), 방문 인원수 (1~10명)
- 등록 완료 시 고유 웨이팅 번호 발급 및 화면 표시
- 같은 브라우저/기기에서 중복 등록 방지 (localStorage 기반 세션 관리)
- **Phase 2 추가:** 매장 웨이팅 상태가 마감(브레이크타임·영업종료·만석)인 경우 등록 불가 안내 표시

### FR-02. 실시간 대기 순서 확인 (Phase 1, 기존 유지)

- 등록 후 대기 현황 페이지로 자동 이동
- 표시 정보: 내 웨이팅 번호, 현재 대기 순서, 앞 대기팀 수
- 실시간 업데이트: SSE 방식으로 순서 자동 갱신 (지연 ≤ 1초)
- 페이지 새로고침 시에도 웨이팅 정보 유지
- **Phase 2 변경:** 예상 대기시간 = 점주 설정 `팀당 평균 이용시간 ÷ 테이블 수 × 앞 대기팀 수`

### FR-03. 점주 회원가입 및 로그인

- 이메일 + 비밀번호 기반 회원가입
- 회원가입 시 매장 기본 정보 입력 (매장명, 업종, 주소, 영업시간)
- 로그인 성공 시 JWT 발급, 이후 요청에 Bearer 토큰 포함
- 로그아웃 시 토큰 무효화
- 비밀번호 분실 대비 이메일 재설정 기능 (Phase 2 범위)

### FR-04. 매장 설정 관리

- 테이블 수 설정 (1~100개)
- 팀당 평균 이용시간 설정 (분 단위, 기본값 30분)
- 대기자 알림 임계값 설정 (예: 10팀 이상이면 점주에게 알림)
- QR 코드 재발급 및 PNG 다운로드
- 매장 정보 수정 (매장명, 영업시간 등)
- 온보딩 가이드 재확인 기능

### FR-05. 실시간 대기 목록 조회 (점주 대시보드)

- 현재 WAITING / CALLED 상태 팀 목록 실시간 표시
- 목록 항목: 대기번호, 이름, 인원수, 등록 후 경과시간
- SSE로 실시간 갱신 (신규 등록·상태 변경 즉시 반영)
- 오늘의 통계 요약: 총 대기 등록 수 / 입장 완료 수 / 노쇼 수 / 취소 수

### FR-06. 웨이팅 처리 (입장·노쇼·호출)

- **호출:** 특정 팀에게 "입장해 주세요" 푸시 알림 발송 (WAITING → CALLED)
- **입장 처리:** 해당 팀 ENTERED 처리, 전체 손님 대기 순서 자동 갱신
- **노쇼 처리:** 응답 없는 팀 CANCELLED 처리, 대기열 갱신
- 처리 시 실수 방지를 위한 확인 다이얼로그 표시
- 처리 완료 후 SSE로 모든 대기 손님에게 순서 갱신 이벤트 발송

### FR-07. 웨이팅 상태 제어 (마감 관리)

마감 상태는 세 가지로 구분하며, 각 상태가 손님 화면에도 실시간으로 표시됩니다.

| 상태     | 설명           | 손님 화면 표시         |
|--------|--------------|------------------|
| 운영 중   | 정상 웨이팅 등록 가능 | 웨이팅 등록 폼 표시      |
| 브레이크타임 | 일시 중단, 재개 가능 | "현재 브레이크타임입니다"   |
| 만석     | 잠시 등록 중단     | "현재 만석입니다"       |
| 영업 종료  | 당일 마감        | "오늘 영업이 종료되었습니다" |

### FR-08. 점주 알림

- 대기자 수가 설정한 임계값 이상이 되면 점주에게 브라우저 푸시 알림 발송
- 손님이 웨이팅 등록할 때마다 대시보드 실시간 갱신 (별도 알림 없이 UI 갱신)
- 알림 수신 여부는 점주가 ON/OFF 설정 가능

### ❌ Phase 2 제외 범위 (Out of Scope)

| 기능              | 목표 Phase |
|-----------------|----------|
| SMS / 카카오 알림 발송 | Phase 3  |
| 직원 계정 및 권한 관리   | Phase 3  |
| 다중 매장 관리        | Phase 3  |
| 웨이팅 순서 수동 조정    | Phase 3  |
| 날짜/시간 예약 기능     | Phase 4  |
| 결제 연동           | Phase 4  |
| 일별/주별 통계 리포트    | Phase 3  |

## 6. 비기능 요구사항 (Non-Functional Requirements)

| 분류  | 요구사항                      | 목표 수치                               |
|-----|---------------------------|-------------------------------------|
| 성능  | QR 스캔 후 웨이팅 페이지 최초 로딩     | ≤ 3초 (LCP 기준)                       |
| 성능  | 실시간 순서 업데이트 지연            | ≤ 1초                                |
| 성능  | 점주 입장 처리 후 손님 화면 갱신       | ≤ 1초                                |
| 가용성 | 서비스 업타임                   | ≥ 99%                               |
| 보안  | JWT 토큰 만료 시간              | Access Token 1시간, Refresh Token 7일  |
| 보안  | 점주 API 비인가 접근 차단          | JWT 검증 실패 시 401 반환                  |
| 보안  | 타 매장 데이터 접근 차단            | storeId 소유권 서버 검증                   |
| 보안  | 비밀번호 저장                   | BCrypt 해싱                           |
| 호환성 | 점주 대시보드 브라우저              | Chrome, Safari 최신 2버전 (데스크탑/모바일 모두) |
| UX  | 점주 회원가입 → QR 발급 완료까지 단계 수 | ≤ 5단계                               |
