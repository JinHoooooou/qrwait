## 1. 제품 개요

QR 웨이팅 서비스는 식당 방문 고객이 **별도 앱 설치 없이** 매장 입구의 QR 코드를 스캔하는 것만으로 웨이팅을 등록하고, 실시간으로 본인의 대기 순서를 확인할 수 있는 웹 기반 서비스입니다.

기존 테이블링(키오스크 기기 필요)이나 캐치테이블(앱 설치 필요) 대비 **진입 장벽을 최소화**하는 것이 핵심 차별점입니다.

| 항목         | 내용                                  |
|------------|-------------------------------------|
| 서비스명       | QR Wait (가칭)                        |
| 서비스 유형     | 웹 애플리케이션 (PWA 고려)                   |
| MVP 타겟 사용자 | 식당 방문 손님 (B2C)                      |
| 핵심 가치      | 앱 설치 없이 QR 스캔만으로 웨이팅 등록 및 실시간 순서 확인 |
| MVP 제외 범위  | 점주용 대시보드, SMS/푸시 알림, 결제, 예약 기능      |

## 2. 문제 정의 (Problem Statement)

### 현재 상황

- 인기 식당에서 손님은 입구에서 수기 명단을 작성하거나 직원을 기다려야 함
- 테이블링 같은 키오스크 솔루션은 기기 구매/렌탈 비용(44만원+)이 발생하고 월 이용료(9.9만원)도 부담됨
- 캐치테이블은 앱 다운로드가 필요하여 고령층·단순 방문객의 진입 장벽이 높음
- 대기 중 손님은 본인 순서를 확인하기 위해 계속 직원에게 묻거나 입구 근처를 서성여야 함

### 우리가 해결하는 문제

- ✅ **손님:** 앱 설치 없이 QR 스캔 한 번으로 웨이팅 등록
- ✅ **손님:** 자리로 이동하거나 자유롭게 이동하면서 실시간으로 대기 순서 확인
- ✅ **점주:** 별도 기기 없이 QR 출력만으로 서비스 도입 가능 (초기 비용 최소화)

## 3. 목표 및 성공 지표

### 제품 목표 (Product Goals)

| 목표         | 세부 내용                        | 우선순위 |
|------------|------------------------------|------|
| 웨이팅 등록 간소화 | QR 스캔 후 3단계 이내로 웨이팅 등록 완료    | P0   |
| 실시간 순서 확인  | 웨이팅 등록 후 내 순서를 실시간으로 확인 가능   | P0   |
| 낮은 진입 장벽   | 별도 앱 설치 없이 브라우저만으로 서비스 이용 가능 | P0   |
| 빠른 로딩      | QR 스캔 후 3초 이내 웨이팅 페이지 로드     | P1   |

### MVP 성공 지표 (KPI)

- 웨이팅 등록 완료율 **≥ 90%** (등록 페이지 접속 후 최종 등록까지 이탈 없음)
- 실시간 순서 확인 페이지 평균 체류 시간 측정 (재방문율 확인)
- 웨이팅 등록 소요시간 **평균 30초 이내**

## 5. 기능 요구사항 (Functional Requirements)

### FR-01. QR 스캔 후 웨이팅 등록

- QR 코드에는 매장 고유 ID가 인코딩되어 있음
    - 예: `https://qrwait.com/wait?storeId=abc123`
- 스캔 시 별도 앱 없이 브라우저에서 웨이팅 등록 페이지로 이동
- 입력 항목: 이름(닉네임), 방문 인원수 (1~10명)
- 등록 완료 시 고유 웨이팅 번호(토큰) 발급 및 화면 표시
- 같은 브라우저/기기에서 중복 등록 방지 (localStorage 기반 세션 관리)

### FR-02. 실시간 대기 순서 확인

- 등록 후 대기 현황 페이지로 자동 이동
- 표시 정보: 내 웨이팅 번호, 현재 대기 순서 (예: 3번째), 앞에 대기 중인 팀 수
- 실시간 업데이트: 앞 팀이 입장할 때마다 순서 자동 갱신 (SSE 방식)
- 페이지 새로고침 시에도 웨이팅 정보 유지 (웨이팅 토큰 기반 조회)
- 전체 대기 현황 표시: 현재 총 대기팀 수, 예상 대기 시간 (평균 입장 간격 기반)

### FR-03. 매장 등록 및 QR 코드 생성

- 매장 이름을 입력하여 신규 매장을 등록할 수 있음
- 등록 시 고유 QR 코드 값(UUID)이 자동 생성됨
- 등록 완료 시 매장 ID, 매장 이름, QR 스캔 URL을 응답으로 반환
    - QR URL 형식: `https://qrwait.com/wait?storeId={storeId}`
- **최소 점주 웹 페이지 (`/owner`) 제공:** 매장명 입력 → 등록 API 호출 → QR 코드 이미지 화면 표시
    - 별도 앱/도구 없이 브라우저에서 QR 코드를 생성하고 인쇄/저장 가능
- REST API(`POST /api/stores`, `GET /api/stores/{storeId}/qr`)로도 직접 호출 가능
- 인증 없음 (MVP 범위)

### ❌ MVP 외 범위 (Out of Scope)

| 기능                             | 목표 Phase |
|--------------------------------|----------|
| 점주용 대시보드 (웨이팅 목록 관리, 입장 처리 버튼) — QR 코드 생성 페이지(`/owner`)는 MVP 포함 | Phase 2  |
| SMS / 카카오 알림 발송                | Phase 2  |
| 회원가입 / 로그인                     | Phase 2  |
| 예약 기능 (날짜/시간 지정)               | Phase 3  |
| 결제 연동                          | Phase 3  |

## 6. 비기능 요구사항 (Non-Functional Requirements)

| 분류  | 요구사항                  | 목표 수치                             |
|-----|-----------------------|-----------------------------------|
| 성능  | QR 스캔 후 웨이팅 페이지 최초 로딩 | ≤ 3초 (LCP 기준)                     |
| 성능  | 실시간 순서 업데이트 지연        | ≤ 1초                              |
| 가용성 | 서비스 업타임               | ≥ 99% (MVP 단계)                    |
